# P3:开源 VLM 自蒸馏方案(规格 §7)

目标:用 L1+L2 规则引擎自动标注的数据 LoRA 微调开源 VLM,覆盖规则 CV 处理不了的
非标准 HUD 场景(OB 视角、无 HUD 片段、非常规分辨率)。规则引擎是教师,VLM 是学生。

## 数据(本仓库已实现)

```bash
vmontage detect                # 先在真实素材上跑出 events.json
vmontage export-vlm-dataset    # → 项目/dataset/{images, labels.jsonl, sft.jsonl}
```

- 标签由规则引擎产出(弱监督):kill_banner / death / headshot / round_end /
  ally_alive / enemy_alive,采样含事件正样本、横幅持续期样本、alive 状态样本与负样本。
- `split=test` 是确定性留出的一致性测试集(settings.yaml `dataset.test_every_n`),
  **训练一律排除**,训练后用它测 VLM↔规则引擎一致率。
- 多个项目的 dataset 可直接合并(labels/sft 均为 JSONL,image 路径各自独立)。
- **职业转播/OB 素材只进 test,不混入训练**(规格红线,防止风格泄漏假高分)。

## 训练(需 GPU 机器,例如用户的 Windows/NVIDIA 或租用)

- 基座:Qwen2.5-VL-7B-Instruct(或资源紧张时 3B)。
- 框架:LLaMA-Factory 或 ms-swift,二者均原生支持 Qwen2.5-VL 的 LoRA SFT,
  `sft.jsonl` 的 messages 格式可直接映射(image 字段换成绝对路径)。
- 起步超参:LoRA r=16 alpha=32,lr 1e-4,epoch 2~3,图像分辨率沿用模型默认;
  单卡 24GB(4090)配 QLoRA 可训 7B。
- 数据量参考:一小时素材约产出 1~2k 样本;建议 ≥10 小时素材再启动首轮训练。

## 验收

1. 一致性测试集上,各字段与规则引擎标签的一致率 ≥95%(kill_banner/death),
   存活人数误差 ≤1(ally/enemy)。
2. 泛化测试:人工标注 50 帧 OB 视角/无 HUD 素材,kill 判定查全查准 ≥80%
   (规则 CV 在这类素材上接近 0,这是 VLM 存在的意义)。
3. 通过后,VLM 作为 detector 的旁路后端接入(L1 输出契约不变,events.json 格式
   完全一致),规则 CV 仍是默认路径。

## 风险

- 弱监督噪声:规则引擎的误检会被学进去 → 训练前先用 scorecards 的 confidence
  过滤低置信样本(labels.jsonl 保留了来源,可回查)。
- 单一玩家 HUD 风格过拟合 → 素材至少混两种分辨率;必要时加数据增广(裁剪/压缩)。
